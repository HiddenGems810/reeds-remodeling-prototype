# Reeds Painting and Remodeling Service — Design Asset Reference Pack

## Goal

Before building the frontend prototype, create a complete design asset direction for **Reeds Renovation Concierge**, the no-backend booking-app prototype for **Reeds Painting and Remodeling Service**.

The visual system should feel like:

* Apple-level cleanliness
* Awwwards-style composition
* premium local craftsmanship
* guided home renovation planning
* polished project concierge, not a basic contractor form

Do not copy any reference source directly. Deconstruct principles, then rebuild them into a custom Reeds-specific system.

---

# 1. Core Visual Thesis

Create a visual world around this idea:

**“Raw home surfaces becoming finished, trusted spaces.”**

The interface should use:

* paint swatches
* blueprint lines
* masked before/after reveals
* wall texture
* plaster/card surfaces
* refined contractor measurement marks
* room-transformation panels
* clean project-summary cards
* tactile but minimal material details

Avoid:

* generic contractor stock-photo websites
* cheap orange/blue handyman colors
* overdone AI gradients
* floating blobs
* fake glassmorphism
* dark-mode-only SaaS styling
* template-looking shadcn cards

## Palette Behavior

* Background should feel bright and clean, not sterile.

* Use texture and spacing more than color noise.

# 4. Typography Asset System

Use a two-font system.

## Recommended Pairing Option A

* Display: Geist / editorial geometric display style
* Body: Inter / clean UI sans

## Recommended Pairing Option B

* Display: Playfair Display or a refined editorial serif
* Body: Geist or Inter

## Typography Rules

* Hero headline should feel confident and architectural.
* Body copy must be highly readable.
* Use clamp-based responsive type.
* Do not use ultra-light body text.
* Avoid centered long paragraphs.
* Use type hierarchy before relying on cards.

## Type Tokens

```css
--font-display: ;
--font-body: ;

--type-hero: clamp(...);
--type-h1: clamp(...);
--type-h2: clamp(...);
--type-h3: clamp(...);
--type-body: ;
--type-small: ;
--type-caption: ;
```

# 11. Animation Asset Direction

Motion signature:

**Editorial Craft Motion**

Use:

* masked headline reveal
* slow material-panel entrance
* subtle service-card lift
* booking step slide/fade
* progress fill animation
* before/after reveal
* section label line-draw animation

Avoid:

* scroll-jacking
* constant parallax
* huge WebGL
* cursor gimmicks
* animation that hides copy
* motion on every element

Timing:

* microinteractions: 120–220ms
* cards/forms: 180–300ms
* section reveals: 350–700ms
* hero motion: under 1600ms
* always respect reduced motion

---

# 12. Section Transition Concepts

Create smooth section transitions between:

## Hero → Services

Paint swatch panels slide into service cards.

## Services → Builder

Selected service chips visually feed into the project brief builder.

## Builder → Before/After

Project brief card transitions into transformation preview.

## Before/After → Process

Reveal slider line becomes the timeline line.

## Process → CTA

Timeline resolves into final consultation CTA.

These transitions should be conceptual, not heavy. The site must still feel fast.

# 15. Quality Bar

The design assets pass only if:

* Reeds looks premium but still approachable.
* The booking flow feels custom, not template-based.
* The visual system clearly belongs to painting/remodeling.
* The interface is clean enough for Apple-style trust.
* The creative components feel intentional, not pasted in.
* The site remains fast, accessible, and mobile-first.
* No fake proof, fake reviews, or unverified claims are used.
